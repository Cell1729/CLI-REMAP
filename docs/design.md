# キーボード・リマップ CLI 設計ドキュメント

## 1. 概要
本プロジェクトは、QMK/VIAプロトコルに対応した自作キーボードに対して、lazygitのようなTUIからリマップを行うツールである。特定のキーボードに依存せず、キーボード定義（JSON）を読み込むことで様々なモデルに対応する汎用型ツールを目指す。

## 2. 技術的背景
### 2.1 HID (Human Interface Device) 通信
キーボードとPCは通常、HIDプロトコルで通信する。
- **標準入力**: 通常のタイピング（文字入力）。
- **Raw HID**: 設定変更や情報取得のために使用される非標準のデータ転送チャネル。

### 2.2 VIA プロトコル
VIA対応のキーボードは、Raw HIDを介して独自のバイナリ命令を受け取る。
- **通信の流れ**:
    1. PCから「特定のコマンドID」を含むデータを送る。
    2. キーボードがそれを受け取り、EEPROM（メモリ）を書き換える。
    3. キーボードから「成功」のレスポンスが返る。

## 3. 主要な仕組み（リマップの仕組み）
Web版の「Remap」がブラウザで行っていることを、CLIツールで再現する。

### 3.1 コマンド例
VIAプロトコルには以下のようなコマンドが存在する。
- `GET_PROTOCOL_VERSION` (0x01): デバイスがVIAに対応しているか確認。
- `DYNAMIC_KEYMAP_SET_KEYCODE` (0x05): 指定したレイヤー・行・列のキーコードを書き換える。
- `DYNAMIC_KEYMAP_GET_KEYCODE` (0x04): 現在設定されているキーコードを読み出す。

### 3.2 必要なライブラリ
- **TUIフレームワーク**: `Textual`
- **キーボード通信 (VIA API)**: `qmk-via-api` (Python bindings)
    - 低レイヤーのHIDパケット構築を自動化し、キーコードの取得・設定を抽象化された関数で実行可能。
- **データ管理**: `Pydantic`

## 4. データ構造：キーボード定義 (JSON)
RemapやVIAで使用されるようなJSON形式をベースに、ツールが盤面を描画し、HID命令を生成するための情報を管理する。
- **Metadata**: VID/PID, キーボード名。
- **Layout**: キーの物理配置 (x, y 座標)。
- **Matrix**: 各キーの行(row)・列(col)情報（HID送信時に使用）。

## 5. TUI インターフェース案（ユーザー体験）
`Textual` を使用して、接続されたデバイスとJSON定義をリンクさせたUIを構築する。

- **Keyboard Selector**: 接続されたHIDデバイスの中から、対応するJSON定義があるのを選択。
- **Sidebar (Layers)**: レイヤー一覧。
- **Main Area (Layout)**: JSONのLayout情報を元に、動的にキーを描画。
- **Bottom Panel (Keycode Picker)**: カテゴリ別のキーコード選択。
- **Status Bar**: デバイス情報と操作ガイド。

## 6. 実装に向けたステップ
1. **環境構築**: Python 仮想環境の作成、`textual`, `qmk-via-api` のインストール。
2. **デバイス認識**: `qmk-via-api` のデバイススキャン機能を使用して接続を確認。
3. **TUIプロトタイプ**: `Textual` で動的なキーレイアウト（JSONベース）を描画。
4. **API統合**: `Textual` の操作から `qmk-via-api` の関数を呼び出し、実機の設定を読み書き。
